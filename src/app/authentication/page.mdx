# Autenticação

Você precisará autenticar suas solicitações para acessar qualquer um dos endpoints na API do Protocol. Neste guia, veremos como funciona a autenticação. O Protocol oferece duas maneiras de autenticar suas solicitações de API: autenticação básica e OAuth2 com um token — OAuth2 é a maneira recomendada. {{ className: 'lead' }}

## Autenticação básica

Com a autenticação básica, você usa seu nome de usuário e senha para autenticar suas solicitações HTTP. A menos que você tenha um bom motivo, provavelmente não deveria usar autenticação básica. Veja como autenticar usando cURL:

```bash {{ title: 'Exemplo de solicitação com autenticação básica' }}
curl https://api.protocol.chat/v1/conversations \
  -u username:password
