# Patient

Os patient são uma parte essencial do sistema administrativo — eles contêm informações detalhadas sobre os doadores e receptores registrados, incluindo detalhes pessoais, endereço e órgãos associados. Nesta página, exploraremos os diferentes endpoints relacionados aos patient que você pode usar para gerenciá-los programaticamente. Veremos como consultar, criar, atualizar e validar patient.

## O modelo de patient

O modelo de patient contém todas as informações sobre os patient cadastrados no sistema. Essas informações incluem dados do usuário, tipo de paciente, tipo sanguíneo, órgãos associados e endereço.

### Propriedades

| Nome               | Tipo    | Descrição                                |
|---------------------|---------|------------------------------------------|
| **id**             | string  | Identificador único do paciente.         |
| **name**           | string  | Nome do paciente.                        |
| **email**          | string  | E-mail do paciente.                      |
| **role**           | string  | Função do paciente no sistema.           |
| **patientType**    | string  | Tipo do paciente (doador/receptor).      |
| **bloodType**      | string  | Tipo sanguíneo do paciente.              |
| **organs**         | array   | Lista de órgãos associados ao paciente. |
| **address**        | object  | Endereço do paciente.                    |
| **hasOrgansSelected** | boolean | Indica se o paciente possui órgãos selecionados. |
