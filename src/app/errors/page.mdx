# Erros

Neste guia, falaremos sobre o que acontece quando algo dá errado enquanto você trabalha com a API. Erros acontecem, e na maioria das vezes, eles serão seus, não nossos. Vamos analisar alguns códigos de status e tipos de erros que você pode encontrar. {{ className: 'lead' }}

Você pode saber se sua solicitação foi bem-sucedida verificando o código de status ao receber uma resposta da API. Se uma resposta não for bem-sucedida, você pode usar o tipo de erro e a mensagem de erro para entender o que deu errado e realizar uma depuração rudimentar (antes de entrar em contato com o suporte).

<Note>
  Antes de entrar em contato com o suporte com um erro, esteja ciente de que 99% de todos os erros relatados são, na verdade, erros de usuário. Portanto, verifique cuidadosamente seu código antes de entrar em contato com o suporte do Protocol.
</Note>

---

## Códigos de status

Aqui está uma lista das diferentes categorias de códigos de status retornados pela API do Protocol. Use esses códigos para entender se uma solicitação foi bem-sucedida.

<Properties>
  <Property name="2xx">
    Um código de status 2xx indica uma resposta bem-sucedida.
  </Property>
  <Property name="4xx">
    Um código de status 4xx indica um erro do cliente — isso significa que o problema é _seu_.
  </Property>
  <Property name="5xx">
    Um código de status 5xx indica um erro do servidor — você provavelmente não verá esses.
  </Property>
</Properties>

## Tipos de erros

<Row>
  <Col>

    Sempre que uma solicitação não for bem-sucedida, a API do Protocol retornará uma resposta de erro com um tipo de erro e uma mensagem. Você pode usar essas informações para entender melhor o que deu errado e como corrigir. A maioria das mensagens de erro é bastante útil e prática.

    Aqui está uma lista dos dois tipos de erro suportados pela API do Protocol — use esses tipos para entender o que você fez de errado.

    <Properties>
      <Property name="api_error">
        Isso significa que nós cometemos um erro, o que é altamente especulativo e improvável.
      </Property>
      <Property name="invalid_request">
        Isso significa que você cometeu um erro, o que é muito mais provável.
      </Property>
    </Properties>

  </Col>
  <Col>

    ```bash {{ title: "Resposta de erro" }}
    {
      "type": "api_error",
      "message": "Não acredito que isso está acontecendo!?",
      "documentation_url": "https://protocol.chat/docs/errors/api_error"
    }
    ```

  </Col>
</Row>
