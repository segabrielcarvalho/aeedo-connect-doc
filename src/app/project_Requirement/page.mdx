## Requisitos do Sistema para o Projeto de Doação de Órgãos

Este documento descreve os requisitos necessários para a implementação do sistema de doação de órgãos. O sistema será dividido em duas partes principais: o **backend**, desenvolvido em **Laravel** com **MySQL**, e o **frontend**, que oferecerá uma interface intuitiva e responsiva. A seguir, detalhamos as principais funcionalidades e a estrutura do sistema.

### Contextualização do Projeto

O Projeto de Extensão de Doação de Órgãos é uma iniciativa de grande relevância social, inicialmente movida pela Professora Mariana Maranhão e abraçada por diversos cursos da UniEVENGÉLICA, incluindo Engenharia de Software. O projeto busca conscientizar a comunidade sobre a importância da doação de órgãos e facilitar o processo de cadastro de doadores.

O Brasil possui o maior sistema público de saúde do mundo, o SUS, e é o segundo país que mais realiza transplantes de órgãos. Contudo, está em 26º lugar em doações, proporcionalmente à população. Entre os fatores que contribuem para esse cenário estão mitos sobre a morte, desinformação e problemas logísticos. A legislação brasileira regula a doação de órgãos desde 1997 (Lei 9.434), exigindo confirmação da morte encefálica por dois médicos independentes e autorização da família. Em março de 2024, foi instituída a AEDO, uma declaração eletrônica de vontade que elimina a necessidade de consulta à família, garantindo que a vontade do doador prevaleça.
#### Objetivos do Projeto

1. Informar e conscientizar sobre a importância da doação de órgãos.
2. Promover discussões científicas e desmistificar o tema.
3. Propor soluções para melhorar o sistema, como campanhas educativas e aprimoramento logístico.
4. Envolver todos os cursos da instituição, explorando como cada área pode contribuir, seja com materiais didáticos, jogos, pesquisas ou campanhas.

### Backend (Laravel e MySQL)

O backend do sistema será implementado utilizando o framework **Laravel** e o banco de dados **MySQL**. A base de dados será composta por várias tabelas interconectadas para garantir um gerenciamento eficiente de usuários, órgãos, hospitais e suas relações.

#### Tabelas Necessárias

- **Usuários**: Armazenará informações pessoais dos usuários, como nome, email e senha. Além disso, conterá os campos `id_endereco` e `id_perfil`, associando o usuário a um endereço e a um perfil específico (Administrador, Receptor ou Doador).
- **Perfil**: Definirá o tipo de usuário no sistema. Cada usuário terá um perfil associado que pode ser Administrador, Receptor (aguardando por um órgão) ou Doador (oferecendo um órgão para doação).
- **Endereços**: Contém dados completos sobre o endereço dos usuários e hospitais, como logradouro, cidade, estado, etc.
- **Órgãos**: Armazena informações sobre os órgãos disponíveis para doação ou aguardando doação. Cada órgão será relacionado a um doador ou receptor, com informações como tipo do órgão, condições e status.
- **Relação Usuários-Órgãos**: Fará a ligação entre os usuários e os órgãos, identificando se o usuário é Doador ou Receptor.
- **Hospitais**: Contém informações sobre os hospitais cadastrados, como nome, localização, especialidades e dados de contato.
- **Hospitais-Usuários**: Relaciona os usuários aos hospitais, permitindo associar médicos ou outros profissionais a um hospital específico.

#### Funcionalidades do Backend

- **Autenticação e Autorização**: O sistema utilizará o **Laravel Passport** para autenticação de usuários. Somente usuários autenticados poderão acessar funções sensíveis, como visualizar dados de doadores e receptores. A autorização será baseada no perfil de cada usuário, garantindo que administradores, receptores e doadores tenham permissões adequadas.
- **Cadastro e Exibição de Dados**: Administradores poderão cadastrar novos usuários, órgãos e hospitais, bem como visualizar e editar dados existentes. Receptores e doadores terão acesso ao seu próprio perfil e aos órgãos relacionados, com informações detalhadas sobre o status de sua doação ou espera.

#### Relacionamento entre Tabelas

O sistema foi projetado para garantir uma estrutura de dados interconectada. Cada tabela se relaciona com outra para possibilitar a navegação eficiente das informações. Por exemplo, a relação entre **usuários** e **órgãos** permitirá identificar facilmente quais órgãos estão disponíveis para doação e quais receptores estão aguardando por eles.

### Frontend

A interface do usuário será desenvolvida para proporcionar uma experiência intuitiva e responsiva, com telas adaptadas a diferentes perfis de usuários. O frontend deve ser de fácil navegação e visualização, permitindo que os usuários encontrem rapidamente as informações necessárias.

#### Principais Telas

- **Tela de Login**: Permitir que os usuários se autentiquem no sistema. Apenas usuários registrados e autenticados terão acesso ao sistema, com segurança garantida por autenticação via **Laravel Passport**.
- **Tela de Cadastro**: Novos usuários poderão se registrar, fornecendo informações como nome, email, endereço e perfil (Doador, Receptor, Administrador).
- **Dashboard**:
  - **Administrador**: Acesso completo ao sistema, com a capacidade de visualizar e gerenciar usuários, hospitais e órgãos. Ele poderá cadastrar novos hospitais e órgãos, gerenciar dados e realizar todas as ações administrativas.
  - **Receptor**: Acesso a um painel dedicado, no qual poderá visualizar os órgãos que está aguardando. Ele também poderá acessar detalhes sobre o status dos órgãos disponíveis para doação.
  - **Doador**: Ver a lista de órgãos que cadastrou para doação. Ele também terá a possibilidade de atualizar ou remover órgãos doados, caso necessário.

### Funcionalidades do Sistema

O sistema de doação de órgãos terá as seguintes funcionalidades principais:

1. **Cadastro e gerenciamento de usuários**, incluindo perfis diferenciados para doadores, receptores e administradores.
2. **Cadastro e gerenciamento de órgãos**, com informações detalhadas sobre cada órgão disponível ou em espera para doação.
3. **Relacionamento entre usuários e órgãos**, para garantir que as doações sejam corretamente atribuídas aos receptores e que os doadores possam gerenciar suas doações.
4. **Cadastro de hospitais**, com a associação de médicos e profissionais aos respectivos hospitais.
5. **Dashboard para visualização de informações**, oferecendo aos administradores e usuários uma visão clara de dados importantes, como listas de órgãos e usuários.
